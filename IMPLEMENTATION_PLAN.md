# Implementation Plan: CodeGenius Upgrade

## Current State

| Area | What it does |
|---|---|
| `express-backend/Routes/bot.routes.js` | Single `/bot/chat` GET — takes a massive prompt string from the frontend, fires it at Gemini, returns plain text |
| `express-backend/Routes/question.routes.js` | Serves pre-seeded static questions from MongoDB |
| `express-backend/Models/Question.model.js` | Simple schema — just `question` + `techStack` |
| `frontend/Interview.tsx` | Fetches all questions at once on mount, records speech, builds a hardcoded prompt string, gets a wall of plain text back, shows it in a `<p>` tag |

**Java backend is already gone** — deleted in a prior commit. No action needed there.  
**All dependencies already installed** — `@google/generative-ai`, `multer`, `pdf-parse`, `openai` are all in `package.json`. Nothing to `npm install`.

---

## Target State

The app becomes **fully session-based and resume-aware**. The user uploads their resume PDF, the backend reads it, and every question + every piece of feedback is dynamically generated by Gemini with full context (resume + conversation history + difficulty level).

---

## Phase 1 — Backend: LLM Service

**File to create:** `express-backend/services/llm.service.js`

Shared utility that wraps Gemini so we only initialize the client once.

- `generateText(prompt)` — calls Gemini, returns a plain string (used for question generation)
- `generateJSON(prompt)` — calls Gemini, strips markdown code fences, parses and returns a JS object (used for evaluation)

---

## Phase 2 — Backend: InterviewSession Model

**File to create:** `express-backend/Models/InterviewSession.model.js`

Mongoose schema to persist the full state of one interview:

```
{
  techStack: String,           // "mern" | "node" | "java"
  resumeText: String,          // full extracted text from the PDF
  currentDifficulty: String,   // "Easy" | "Medium" | "Hard"
  history: [                   // array grows with each Q&A round
    {
      question: String,
      answer: String,
      score: Number,
      strengths: [String],
      improvements: [String]
    }
  ],
  status: String,              // "active" | "completed"
  createdAt: Date
}
```

Every call to generate the next question loads the session and passes the full history to Gemini, so questions never repeat and difficulty adapts based on scores.

## Phase 3 — Backend: Interview Routes

**File to create:** `express-backend/Routes/interview.routes.js`

### `POST /api/interview/start`
- Uses `multer` to accept a PDF file upload + `techStack` in the request body
- Uses `pdf-parse` to extract text from the PDF
- Creates a new `InterviewSession` in MongoDB (saves `resumeText`, `techStack`, difficulty starts at `"Easy"`)
- Returns: `{ sessionId, question: "Please introduce yourself and walk me through your background." }`

### `POST /api/interview/evaluate`
- Body: `{ sessionId, question, answer }`
- Fetches the session from DB
- Builds a prompt asking Gemini to return ONLY valid JSON: `{ score, strengths, improvements }`
- Adjusts difficulty: score ≥ 7 → bump up, score ≤ 4 → bump down
- Saves the Q&A round into `session.history`
- Returns: `{ score: 8, strengths: ["..."], improvements: ["..."] }`

### `POST /api/interview/next-question`
- Body: `{ sessionId }`
- Fetches the session (with full history)
- Builds a context-aware prompt: resume + history + current difficulty
- Calls `generateText()` from the LLM service
- Returns: `{ question: "Explain how the Node.js event loop works..." }`

## Phase 5 — Frontend: Rewrite `Interview.tsx`

The component will have **3 distinct phases** controlled by a `phase` state variable.

### Phase 0 — Resume Upload
- Display the tech stack name
- `<input type="file" accept="application/pdf" />`
- "Start Interview" button
- On submit: POST `/api/interview/start` with FormData → save `sessionId` + first question → move to Phase 1

### Phase 1 — Interview Question
- Display current question (from state, not a DB array)
- Webcam + speech recognition
- "Submit Answer" button
- On submit: POST `/api/interview/evaluate` with `{ sessionId, question, answer }` → move to Phase 2

### Phase 2 — Structured Feedback
Replaces the plain text `<p>{feedBack}</p>` with a proper UI:
- **Score** — large bold number `/10`, color-coded (red < 5, yellow 5–7, green ≥ 8)
- **Strengths** — green bulleted list
- **Improvements** — red/orange bulleted list
- "Next Question" button → POST `/api/interview/next-question` → back to Phase 1

### What gets removed from Interview.tsx:
- `useEffect` that fetches `/questions/get`
- `questions` array state + `currentIndex` state
- `feedBack` string state
- axios call to `/bot/chat`
- `handleNextQuestion` / `handleprevious` (replaced by API-driven navigation)

### What gets added:
- `phase` state: `"upload" | "question" | "feedback"`
- `sessionId` state (string)
- `currentQuestion` state (string)
- `feedback` state: `{ score, strengths, improvements }` object
- Resume file state
